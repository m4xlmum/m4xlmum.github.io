Centos运维经验

docker相关的经验

直接对docker的操作

安装docker

sudo yum install -y yum-utils  device-mapper-persistent-data  lvm2
yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
dnf install https://mirrors.aliyun.com/docker-ce/linux/centos/7/x86_64/stable/Packages/containerd.io-1.2.13-3.1.el7.x86_64.rpm
dnf install docker-ce
sudo yum install docker-ce docker-ce-cli
sudo systemctl start docker
docker --version

开机自启

sudo systemctl enable docker

更换源

  • docker中国官方

    https://registry.docker-cn.com

  • 网易

    http://hub-mirror.c.163.com

  • USTC

    http://docker.mirrors.ustc.edu.cn

  • 阿里云　　

    http://<你的ID>.mirror.aliyuncs.com
换源的方法是编辑/etc/docker/daemon.json（如果不存在则创建）:
{
    "registry-mirrors":[
         "http://docker.mirrors.ustc.edu.cn",
         "http://hub-mirror.c.163.com",
         "http://registry.docker-cn.com"
    ] ,
    "insecure-registries":[
       "docker.mirrors.ustc.edu.cn",
         "registry.docker-cn.com"
    ]
}

重启docker

sudo systemctl restart docker

开启docker

systemctl start docker   # 一般开启docker

对docker容器的操作

复制文件到docker容器

sudo docker cp 源vps绝对路径 容器ID:容器内的目标绝对路径

安装CTFd

sudo docker run --name ctfd -d -p 80:8000 ctfd/ctfd

查看端口占用进程

sudo lsof -i: 端口

常用小工具

hexo

解决hexo环境变量失效的问题

hexo的环境变量应为

C:\Users\Administrator\AppData\Roaming\npm\node_modules\hexo-cli\bin

Deepin使用经验

1.在Deepin上重新配置Hexo环境

安装node 和 npm

npm

下面是失败案例，请自动跳过

这个我看着太复杂了，就没有采用。

sudo apt-get update

nodejs.cn中文网上面下载最新版的nodejs的linux版本，随后安装

成功案例

https://www.cnblogs.com/cuihuale-fe/p/15354818.html


// 安装 nodejs
sudo apt-get install nodejs

// 安装 npm
sudo apt-get install npm

// node 版本
node -v
10.8.2

升级node版本

// n 模块是管理 nodejs 版本的
sudo npm install n -g

// 1. 安装稳定版 nodejs
sudo n stable 

// nodejs 版本
node -v
14.17.6

// 2. 安装最新版 nodejs
sudo n latest

// nodejs 版本
node -v 
16.10.0

我直接下载最新版的node了哈哈。但是中间会有一个问题，就是从nodejs.org下载的时候，速度真的是太慢了，所以这里其实是可以改一下这个版本管理工具n的一些配置的。

N_NODE_MIRROR=${N_NODE_MIRROR:-${NODE_MIRROR:-https://nodejs.org/dist}} 改成
N_NODE_MIRROR=${N_NODE_MIRROR:-${NODE_MIRROR:-https://npm.taobao.org/mirrors/node}}

速度马上就上去了

安装hexo

sudo npm install -g hexo-cli

配置hexo

ssh-keygen -t rsa
#之后添加公钥文件id_rsa.pub的内容到github的setting的key中即可

git config --global user.email "2994016494@qq.com"
git config --global user.name "m4xlmum"

这就把这个hexo的环境配好了，更新云端文件只需要使用命令hexo clean && hexo g -d即可。

GitPage配置域名解析

这个可以做做，整个CNAME的域名解析应该可以。

先整个CNAME的插件

https://github.com/leecrossley/hexo-generator-cname

然后在该页面添加你的域名，如blog.shuqingshi.top

最后阿里云的域名解析界面再加上一个CNAME记录即可成功解析。

还不错。
