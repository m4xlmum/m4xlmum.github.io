前言

想做这个东西很久了，主要是想尝试一下MVC的网站开发模式，看看网站从无到有的开发的流程。今天开始可以开始自己实现一个了，我会把开发的过程记在这个文档里面，留作以后的参考。

注:

  1. 本系统暂时只会实现个人发表博客，登录功能主要是用于发表文章更新时用的，相当于一个后台的admin管理界面。

一、前端布局

这里我主要想参考的是下面这个hexo的博客的布局来搞，就是一个初级的练手的项目，不想搞得太深，没什么必要也。

就是最上面是一个导航栏，再往下一点有个轮播图，再向下是个三栏式的博客，左边和右边是小工具的地方，中间放文章的标题与摘要，可以设置封面，不设置封面的话自动加载默认的图片。

二、数据库设置

两个表，一个表中存放用户的相关信息，另一个表中存放发表的博客文章的相关信息。

1.用户信息表

用户名、密码、邮箱

CREATE TABLE [dbo].[userinfo]
(
	[Id] INT NOT NULL PRIMARY KEY, 
    [username] VARCHAR(10) NOT NULL, 
    [password] VARCHAR(20) NOT NULL, 
    [email] VARCHAR(50) NOT NULL
)

2.文章相关表

文章id(唯一)、文章cover、文章标题、文章路径(并不准备把文章存在数据库里面)。

CREATE TABLE [dbo].[AticleInfo] (
    [Id]           INT           NOT NULL,
    [articlecover] VARCHAR (200) NULL,
    [articletitle] VARCHAR (50)  NOT NULL,
    [articlepath]  VARCHAR (200)   NOT NULL,
    [articlesummary] VARCHAR(200) NOT NULL, 
    [articlewordnum] INT NULL, 
    [articlesticky] INT NULL, 
    PRIMARY KEY CLUSTERED ([Id] ASC)
);

把读文章做成接口

这里我把从文件里面读取文章的功能做成了一个接口，把接口就写在了HomeController里面了，未来如果有需要的话，可能会把接口挪到专门的ApiController里面，做成一个专门的接口控制器

//API
public ActionResult APIGetArticle(int? id)
{
    AticleInfo articleInfo = db.AticleInfo.Find(id);
    string path = articleInfo.articlepath;
    path = Request.MapPath(path);  //转换为相对路径

    string readContents = System.IO.File.ReadAllText(path, Encoding.UTF8);
    return Content(readContents);
}

四、文章撰写

​ 使用插件editormd，这里有两种选择，第一种是把editor组件的代码下载到本地，第二就是使用在线的镜像，这里为了稳定采用第一种方案。

​ 先把代码下载下来，下载地址，然后把里面的文件解压到一个文件夹里面，文件夹这里我命名为了editormd，因为Script文件夹里面的文件太多了，非常讨厌，所以这里我把这个插件放在了Content目录下面了。

​ 使用方法也很简单，这里直接把他这个插件的官方网站上的代码修改一下即可使用到我们的ASP项目

@{
    Layout = "~/Views/Shared/_AdminLayout.cshtml";
}
<html>
<head>
    <link rel="stylesheet" href="../Content/editormd/css/editormd.css" />
    <script src="https://cdn.bootcss.com/jquery/1.11.3/jquery.min.js"></script>
    <script src="~/Content/editormd/editormd.min.js"></script>
    <script type="text/javascript">
        $(function () {
            var editor = editormd("editor", {
                // width  : "100%",
                // height : "100%",
                path: "../Content/editormd/lib/"
            });
        });
    </script>
</head>
<body>
    <div id="editor">
        <textarea style="display:none;">
            ### 关于 Editor.md
            **Editor.md** 是一款开源的、可嵌入的 Markdown 在线编辑器（组件），基于 CodeMirror、jQuery 和 Marked 构建。
        </textarea>
    </div>
</body>
</html>

五、权限鉴别

​ 终于写到鉴权的部分了，这个是网站的重中之重啊，稍不注意就是个高危漏洞。。。

​ 这里我准备采取ASP的cookie的鉴权方式，并且设定一个cookie的过期时间，也是为了防止部分xss和ssrf。emmm，这个鉴权的第一个部分呢，就是login的那里了，肯定是要在这里设置初始的cookie的，但是先是需要处理login是否成功。

登录判定(确认密码)

public ActionResult Login(string username, string password)
{
    string truepass = "";
    foreach (var pass in db.userinfo.Where(user => user.username == username))
    {
        truepass = pass.password;
    }
    if (password == truepass)
    {
        string page = Url.Action("Index", "Admin");
        return Redirect(page);
    }
    else
    {
        ViewBag.IsLoginError = true;
        return View();
    }

}

这里设置了一个ViewBag.IsLoginError的值，代表的是是否登陆失败，true表示登陆失败，当登录失败时，html上面做出一个提示，代码如下。

            $("small").text(--timeVal + " 秒后消失")
            if (timeVal <= 0) {
                clearInterval(timer);
            }
        }, 1000);
    });
</script>

                </div>

                </div>
            </div>
        }
    }
}

六、注册用户名唯一

这个的逻辑跟上面的login的密码判断的逻辑是相似的，所以代码我直接copy出来了，稍加改动即可。

Controller

[HttpPost]
public ActionResult Register([Bind(Include = "username,password,email")] userinfo userinfo)
{
    if (ModelState.IsValid)
    {
        userinfo.Id = db.userinfo.Count();
        //验证username的唯一性，如果username已存在则不给添加
        foreach (var name in db.userinfo.Where(user => user.username == userinfo.username))
        {
            ViewBag.IsRegistered = true;
            return View(userinfo);
        }

        db.userinfo.Add(userinfo);
        db.SaveChanges();
        return RedirectToAction("Login");
    }

    return View(userinfo);
}
