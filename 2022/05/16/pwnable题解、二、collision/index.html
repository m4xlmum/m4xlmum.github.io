这个题目，需要让hashcode这个全局变量的值等于check_password()函数返回的值，那我们再去看看这个函数。我觉得这个题目最难理解的点就是下面这句代码int* ip = (int*)p;，我们分析一下这个人难点。

1、难点分析

这个p的数据类型是char*，也就是一个指针，指向的是一个字符数组这个样子。然后(int*)p就对p的值进行了一个强制类型转换，根据我的玄学看法，这个语句的大概意思就是：

把指针指向的char类型的数据强制转换成了int类型。譬如：若p[0]为A，那ip[0]就为0x41

p指向的数据正是passcode的值，也就是我们运行软件的时候输入的参数的值。p的位数为20位，5个ip的值相加为res的最终值。每个ip的最后的值为四个char的值排在一起的值。

也就是说当我们输入20个A的时候，每个ip的值都为0x41414141，验证一下。

然后解释一下这个eax的值，eax寄存器应该是作为，c语言返回值临时存储的地方的。这里出现$1 = 1179010629似乎是因为没有格式化输出，那个太麻烦了，其实这个eax寄存器内的值应该就是1179010629 => 0x46464645了。本来应该是0x146464645的似乎，因为0x41414141 * 5 == 0x146464645，但是不知道为什么这个第九位的1没了，原谅菜鸟吧！

那很容易就得出了这个结果，我们需要把我们输入的passcode每四个为一组，五组加一块的值为hashcode的值。
